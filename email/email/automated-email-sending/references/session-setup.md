# Session-Specific Email Setup (May 2026)

This reference captures the exact configuration from the initial bot email setup
for this environment. Use as a template for new setups.

## Stored Credentials

| What | Where |
|------|-------|
| Gmail address | Vault: `EMAIL_ADDRESS` |
| Gmail App Password | Vault: `EMAIL_PASSWORD` |
| Cloudflare API token | Vault: `CLOUDFLARE_API_TOKEN` (`cfut_` prefixed) |
| Cloudflare Zone ID | Vault: `CLOUDFLARE_ZONE_MONTYGROUP` |
| Cloudflare Account ID | Vault: `CLOUDFLARE_ACCOUNT_ID` |

## DNS State

- MX: `route1.mx.cloudflare.net` (prio 95), `route2.mx.cloudflare.net` (prio 46), `route3.mx.cloudflare.net` (prio 49)
- SPF: `v=spf1 include:_spf.mx.cloudflare.net ~all`
- DKIM: auto-generated by Cloudflare (selector `cf2024-1`)
- Apex: A records to `76.76.21.21` and `76.76.21.98` (Vercel, proxied)
- No CNAME at apex (removed due to MX conflict)

## Email Routing Rule

- `hermes@montygroup.uk` → forwarded to the user's email inbox
- Set up via Cloudflare Dashboard → Email → Email Routing → Add route

## SMTP Relay

- Provider: Gmail SMTP
- Host: `smtp.gmail.com:587` (STARTTLS)
- Script: `~/scripts/hermes-email.py`
- msmtp config: `~/.msmtprc`

## Sending Semantics

- From header: `Hermes Bot <hermes@montygroup.uk>`
- Envelope sender: `hermes@montygroup.uk`
- Authenticated as: the Gmail address (App Password)
- Default recipient for tests: `matthogarth@hotmail.com`
